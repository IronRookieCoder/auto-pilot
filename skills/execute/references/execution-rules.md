# 执行规则

> 本文件定义里程碑执行阶段的核心规则。编排者和子代理都必须遵守。

## 范围纪律

- **严禁范围蔓延**：只做 milestones.json 中定义的里程碑内容，不添加"顺便改进"
- **有疑问先记录**：遇到规格歧义时，返回"阻塞"状态，由编排者或用户决定
- **一次一个里程碑**：完成当前里程碑并通过验证后，才能开始下一个

## 上下文隔离纪律

- **每个里程碑用子代理执行**：通过 Agent 工具为每个里程碑生成独立子代理，完成后上下文自动释放
- **编排者不写代码**：编排者只负责调度里程碑、更新状态文件；代码编写和测试由子代理完成
- **文件是唯一的状态传递机制**：里程碑之间通过 `.workflow/` 文件传递状态，不依赖上下文中的对话历史

## 记录纪律

- **每个里程碑完成后更新 milestones.json**：标记完成状态、记录决策
- **验证结果写入 verify.json**：不依赖自由文本
- **事件追加到 events.jsonl**：关键节点留痕
- **失败也要记录**：验证失败的原因和修复过程都是有价值的信息

## 禁止事项

1. ❌ 不修改 spec.md（除非用户明确要求）
2. ❌ 不跳过验证步骤
3. ❌ 不在里程碑范围外添加功能
4. ❌ 不删除或修改现有测试用例以通过验证
5. ❌ 不忽略 lint/typecheck 错误
6. ❌ 不硬编码应该配置化的值
