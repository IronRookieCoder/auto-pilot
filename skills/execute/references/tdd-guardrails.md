# TDD 铁律

> 本文件定义 TDD 驱动开发的不可违反规则。子代理执行里程碑时必须遵守。

## 核心循环

```
编写测试用例 → 运行测试确认 RED（全部失败）→ 编写实现代码 → 运行验证 →
  ├─ 通过 → 完成
  └─ 失败 → 修改实现代码（禁止改测试）→ 重新验证（最多 3 轮）
```

## 不可违反规则

### 允许的操作

- ✅ 修改实现代码来通过测试
- ✅ 修改实现代码来通过 lint/typecheck
- ✅ 添加新的实现文件
- ✅ 在实现代码中重构以满足要求

### 禁止的操作

- ❌ 修改测试文件中的断言或预期值
- ❌ 删除或跳过测试用例（`.skip()` / `@pytest.mark.skip`）
- ❌ 放宽正则/匹配条件以适配实现
- ❌ 修改 lint 规则配置以消除警告
- ❌ 在 `tsconfig.json` 中跳过类型检查
- ❌ 在里程碑范围外添加功能

### 唯一例外

如果测试用例本身存在明显的 bug（如拼写错误、逻辑错误）：

1. **不要自行修复测试**
2. 在返回结果中标注"测试疑似 bug"
3. 等待编排者或用户确认后才能修改测试

## 修复轮次限制

- 单个验证步骤最多 3 轮修复
- 3 轮后仍失败：返回"阻塞"状态和失败详情
