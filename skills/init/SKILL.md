# 初始化项目记忆

在当前项目根目录创建 `.workflow/` 目录和 5 个记忆文件，建立 AI 编码代理的持久化"外部大脑"。

## 执行步骤

### 1. 前置检查

- 确认当前工作目录是项目根目录（存在 package.json / pyproject.toml / go.mod / Cargo.toml 等项目标识文件，或用户明确指定）
- 如果 `.workflow/` 已存在，提示用户：目录已存在，是否要重新初始化（会覆盖现有文件）

### 2. 收集项目信息

通过以下方式收集（优先自动检测，不足时询问用户）：

| 信息项 | 自动检测方式 | 必需 |
|--------|-------------|------|
| 项目名称 | package.json 的 name / 目录名 | 是 |
| 技术栈 | 检测 package.json / pyproject.toml / go.mod 等 | 是 |
| 项目目标 | 询问用户或从用户输入提取 | 是 |
| 验证命令 | 从 package.json scripts / Makefile 等提取 | 否 |
| 现有目录结构 | 自动扫描 | 否 |

### 3. 创建记忆文件

读取 `references/` 目录下的 5 个模板文件，用收集的项目信息填充，生成：

```
.workflow/
├── spec.md            # 规格说明 - 冻结的项目目标和约束
├── plan.md            # 里程碑计划 - 执行蓝图（初始为空骨架）
├── implement.md       # 执行指令 - AI 代理的操作规程
├── documentation.md   # 状态仪表盘 - 当前状态和活跃问题（保持精简 ≤100 行）
└── changelog.md       # 变更日志 - 完整执行历史归档（日常不读取）
```

**填充规则：**
- `spec.md`：填入项目名称、技术栈、目标描述、约束条件
- `plan.md`：只生成骨架，里程碑部分标记为"待 /auto-pilot:plan 生成"
- `implement.md`：填入验证命令、检测到的项目约定
- `documentation.md`：填入初始化时间、项目基本信息
- `changelog.md`：填入初始化记录

### 4. 输出结果

向用户展示：
1. 创建的文件列表
2. 下一步指引：
   - "请打开 `.workflow/spec.md` 审阅并完善，确保项目目标、范围和约束准确"
   - "重点检查验收标准：每条标准必须可测试（能用自动化测试或验证命令判定通过/失败）"
   - "完善后运行 `/auto-pilot:plan` 生成里程碑计划"

## 重要约束

- **不要猜测项目目标** - 如果用户没有提供需求描述，必须询问
- **模板内容必须从 references/ 读取** - 不要硬编码模板
- **保留用户已有的 .workflow/ 内容** - 如果部分文件已存在且有用户自定义内容，提示用户而非覆盖
- **spec.md 是"冻结"文件** - 在初始化后应该由用户确认，后续流程不应随意修改
